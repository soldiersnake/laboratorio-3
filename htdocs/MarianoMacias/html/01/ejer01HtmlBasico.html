Seccion de etiquetas semanticas estructurales

  • HEADER
  • FOOTER
  • SECTION
  • ARTICLE
  • NAV
  • ADDRESS

Las etiquetas semanticas estructurales estan relacionadas directamente con el significado del contenido y no con los estilos ni las posiciones deltro del documento.

Reglas para etiquetas semanticas:

  • Los distintos tipos de contenido se pueden asociar a las etiquetas SECTION
  • Las unidades de contenido se asocian a las etiquetas ARTICLE. O sea que un SECTION anida a multiples ARTICLES
  • Las etiqueta HEADER Y FOOTER pueder ser anidades dentro de otras HEADER O FOOTER. Osea un FOOTER podria anidar a otro HEADER Y FOOTER
  • La etiqueta NAV debe ser la unica en el documento y corresponde a los elementos de navegacion del documento
  • Reservar etiqueta h1 para usar dentro de etoquetas top level como main, header o footer

  7. Explicar cuando usarían etiquetas semánticas y con que propósito.

    -Las etiquetas semánticas en HTML se utilizan para proporcionar información adicional sobre la estructura y el significado del contenido en una página web. Estas etiquetas permiten a los motores de búsqueda y a los navegadores web comprender mejor la información del sitio y su organización, lo que puede ayudar a mejorar la accesibilidad, el SEO y la experiencia del usuario.

  9. ¿Que diferencia existe entre la etiqueta body de un documento HTML y el body del protocolo HTTP?

    ---La etiqueta < body> en un documento HTML es un elemento que se utiliza para definir el contenido visible de una página web, como el texto, las imágenes y otros elementos que se muestran en el navegador. ---Por otro lado, el término "body" en el contexto del protocolo HTTP se refiere a la parte del mensaje HTTP que contiene los datos enviados al servidor o recibidos del servidor en respuesta a una solicitud HTTP. En otras palabras, el "body" en el protocolo HTTP se refiere a los datos que se envían o reciben a través de una conexión HTTP, ej formulario.
